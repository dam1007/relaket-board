# 프로젝트 분석 및 개선 제안 프롬프트

다음은 현재 Next.js 기반 게시판 프로젝트의 코드베이스를 분석하고 개선 사항을 제안하기 위한 프롬프트입니다. 각 영역별로 현재 상태를 평가하고, 구체적인 개선 방안을 제시해주세요.

## 프로젝트 개요

- **유형**: Next.js 기반 게시판 애플리케이션
- **주요 기술**: Next.js, React, TypeScript, vanilla-extract (CSS-in-JS), Knex.js, MySQL, Redis

## 분석 및 개선 제안 영역

각 영역에 대해 현재 프로젝트의 상태를 평가하고, 발견된 문제점과 함께 구체적이고 실행 가능한 개선 방안을 제시해주세요.

### 1. 코드 품질 및 가독성

- **평가**: 코드의 일관성, 가독성, 주석의 적절성, 중복 코드 여부 등을 평가해주세요.
- **개선 제안**: 코드 컨벤션 강화 (ESLint, Prettier 설정), 리팩토링 필요 영역, 불필요한 코드 제거, 주석 및 문서화 개선 방안 등을 제시해주세요.

### 2. 성능 최적화

- **평가**: 페이지 로딩 속도, API 응답 시간, 이미지 최적화, 번들 크기 등을 평가해주세요.
- **개선 제안**: 이미지 최적화 (Next/Image 사용), 데이터 페칭 전략 (SSR, SSG, CSR), 캐싱 전략 (Redis 활용), 번들 최적화, 불필요한 렌더링 방지, 데이터베이스 쿼리 최적화 방안 등을 제시해주세요.

### 3. 보안

- **평가**: 사용자 인증/인가, 데이터 유효성 검사, XSS/CSRF 방어, 환경 변수 관리, 의존성 취약점 등을 평가해주세요.
- **개선 제안**: 인증/인가 로직 강화, 입력값 검증 강화, 보안 헤더 설정, 의존성 업데이트, 비밀 정보 관리 방안 등을 제시해주세요.

### 4. 유지보수성 및 확장성

- **평가**: 모듈화 정도, 컴포넌트 재사용성, 아키텍처의 유연성, 테스트 용이성 등을 평가해주세요.
- **개선 제안**: 컴포넌트 분리 및 재사용성 증대, 서비스/유틸리티 계층 분리, 타입 정의 개선, 테스트 코드 작성 전략, 새로운 기능 추가 시의 확장성 고려 방안 등을 제시해주세요.

### 5. 사용자 경험 (UX) 및 접근성

- **평가**: UI/UX 일관성, 반응형 디자인, 에러 처리 방식, 로딩 상태 피드백, 웹 접근성 (ARIA) 등을 평가해주세요.
- **개선 제안**: UI/UX 개선 (디자인 시스템 도입), 반응형 디자인 강화, 사용자 친화적인 에러 메시지, 로딩 스피너/스켈레톤 UI 적용, 웹 접근성 가이드라인 준수 방안 등을 제시해주세요.

### 6. 개발 환경 및 워크플로우

- **평가**: 개발 환경 설정의 용이성, 빌드/배포 프로세스, CI/CD 파이프라인 유무, 개발자 경험 등을 평가해주세요.
- **개선 제안**: 개발 환경 설정 자동화, 빌드 시간 단축, CI/CD 도입/개선, 개발자 도구 활용 방안 등을 제시해주세요.

## 제안 형식

각 영역별로 다음 형식에 맞춰 제안해주세요.

```
### [영역 이름]

**현재 상태 평가:**
- [평가 내용 1]
- [평가 내용 2]

**개선 제안:**
- [구체적인 개선 방안 1]
  - [세부 내용/예시]
- [구체적인 개선 방안 2]
  - [세부 내용/예시]
```

이 프롬프트를 사용하여 프로젝트를 심층적으로 분석하고, 실질적인 개선 방안을 도출해주세요.
